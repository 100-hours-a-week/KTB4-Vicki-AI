# 5주차 과제 회고
## 1. 과제 목표
이번 과제에서는 CIFAR-10 데이터셋을 활용하여 ResNet50과 VGG16을 학습시키고 성능을 비교해보았다.

또한 전이학습(Transfer Learning)과 Fine-Tuning의 차이를 직접 실습하며, 모델 학습 과정에서 발생하는 과적합과 과소적합 문제를 분석하고 해결 방법을 적용해 보는 것을 목표로 했다.

과제는 먼저 CIFAR-10 데이터셋을 전처리하고, 
1. ResNet50 으로 Full Fine-Tuning을 진행하고,
2. VGG16으로 Feature Extraction을 진행하였다.
3. 그리고 ResNet50과 VGG16 둘 다 Partial Fine-Tuning을 진행하고 결과를 비교해보았다.

## 2. 배운점
이번 과제를 진행하면서 단순히 모델을 학습시키는 것보다 학습 결과를 분석하는 과정이 더 중요하다는 것을 느꼈다.

ResNet50을 Full Fine-Tuning으로 학습했을 때 학습 성능은 높게 나왔지만 검증 성능과 큰 차이가 발생하며 과적합이 나타났다.
이를 통해 데이터 수가 충분하더라도 모델의 크기와 학습 방식에 따라 과적합이 발생할 수 있다는 점을 알게되었다.

또한 VGG16 Feature Extraction 방식은 대부분의 가중치를 고정하고 분류기만 학습하기 때문에 학습 속도는 빠르지만 충분한 성능을 내지 못할 수 도 있다는 것도 알게되었다.

특히 과적합과 과소적합을 실제 Loss 그래프와 Accuracy 결과를 통해 직접 확인하고 해결방법을 찾아 적용해본것이 가장 큰 배운점인 것 같다.

## 3. 어려웠던점
가장 어려웠던 부분은 두 모델을 비교할 때 공정한 비교를 위해 어떤 조건을 통제해야 하는지 모르겠던 점이었다.

ResNet50의 과적합을 막기 위해 데이터 증강, Early stopping 적용과 epoch 수 줄이기 L2 정규화 등을 적용하여 비교적 좋은 결과를 얻었는데 같은 데이터셋으로 학습한 VGG16은 오히려 과소적합이 발생했다. 

그 원인을 고민해본 결과 ResNet50의 과적합을 방지하고 학습 시간을 줄이기 위해 epoch를 너무 적게 설정하고 학습 데이터에 데이터 증강을 너무 심하게 적용하여 VGG16에서 과소 적합이 발생한 것 같았다.

이 과정에서 서로 다른 구조를 가진 두 모델을 공정하게 비교하는 것이 생각보다 까다로운 작업이라는 것을 알게 되었다. 

또한 CIFAR-10 다운로드 과정에서 403 에러가 계속해서 발생하는 문제도 있었다.

## 4. 해결 과정
ResNet50을 Full Fine-Tuning 했을때 발생한 과적합 문제를 해결하기 위해 Early Stopping, L2 정규화, 데이터 증강 등의 방법을 적용하였다.
그 결과 ResNet50의 과적합을 해결할 수 있었다.

CIFAR-10 다운로드 문제는 기존 방식 대신 Google Drive에 저장된 데이터를 직접 불러오는 방식으로 해결하였다.

## 5. 개선점 및 다음 목표
이번 과제에서는 ResNet50에서 발생한 과적합 문제를 데이터 증강, Early Stopping, Epoch 조정, L2 정규화 등을 통해 어느 정도 개선할 수 있었다. 
하지만 이러한 설정을 동일하게 적용한 VGG16에서는 오히려 과소적합이 발생하여 두 모델 모두에서 적절한 성능을 내는 학습 환경을 찾지는 못했다.

다음에는 데이터 증강 강도, Epoch 수, 학습 가능한 레이어 범위 등을 하나씩 변경하며 추가 실험을 진행해보면 좋을 것 같다. 
특히 현재 VGG16의 과소적합이 데이터 증강의 영향인지, Epoch 부족 때문인지, 또는 모델 구조의 차이 때문인지 확인하고 싶다.
